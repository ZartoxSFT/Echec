Jeu d'Échecs

Cette documentation couvre l'implémentation d'un jeu d'échecs en Java, utilisant une architecture MVC (Modèle-Vue-Contrôleur) et plusieurs patterns de conception.

Structure du Projet

Package model

Contient les classes représentant les éléments du jeu :

  • Pièces (Roi, Reine, Tour, Fou, Cavalier, Pion)
  • Plateau de jeu
  • Mouvements et règles du jeu

Package view

Gère l'interface utilisateur :

  • Interface graphique (UI)
  • Interface console (ConsoleUI)
  • Timer pour les parties chronométrées

Package controller

Gère la logique du jeu :

  • Core : contrôleur principal
  • AI : intelligence artificielle avec plusieurs niveaux

Patterns de Conception Utilisés

  • MVC : séparation des responsabilités entre Modèle, Vue et Contrôleur
  • Strategy : pour les mouvements des pièces
  • Decorator : pour combiner les mouvements (ex: Reine = Tour + Fou)
  • Observer : pour la mise à jour de l'interface utilisateur

Fonctionnalités Principales

  • Toutes les règles des échecs (incluant les coups spéciaux)
  • Interface graphique et console
  • IA avec plusieurs niveaux de difficulté
  • Sauvegarde et chargement des parties
  • Timer pour les parties chronométrées
  • Détection de l'échec, mat et pat

Sérialisation

Le jeu implémente la sérialisation pour permettre la sauvegarde et le chargement des parties. Les classes principales implémentent l'interface Serializable et gèrent correctement les références circulaires et les champs transient.
